##### 14.06.2026
1. Отправлять в GitHub свои коммиты Alembic(а)  
   - При `git pull` второго разработчика, он должен иметь возможность применить полный цикл изменений с БД
   - Чтобы это было возможно - надо пушить папку `alembic/versions` в GitHub
   - Второй разработчик после получения всех изменений через `git pull` должен выполнить 1 команду - `alembic upgrade head` и все миграции в папке `alembic/versions` применяется по умолчанию
   - Первый разработчик должен создать эти миграции через `alembic revision --autogenerate -m <comment>`
